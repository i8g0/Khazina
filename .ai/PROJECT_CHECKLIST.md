# Khazina — AI Pre-Task Checklist

Execute this checklist mentally before every task.

---

- [ ] Read [AI_PROJECT_PROFILE.md](AI_PROJECT_PROFILE.md)
- [ ] Determine task type (documentation, frontend, backend, database, infrastructure, bug fix, refactor, architecture, testing)
- [ ] Read required skills (see [Quick Task Routing](AI_PROJECT_PROFILE.md#quick-task-routing) in profile)
- [ ] Read only required documentation — do not load every doc
- [ ] Verify scope: list files to modify, files NOT to modify, and risks
- [ ] Preserve architecture — no folder moves, framework changes, or new dependencies without approval
- [ ] Prefer minimal changes — smallest diff, no full file rewrites unless requested
- [ ] Update documentation if necessary — only the owning document
- [ ] Validate changes — run relevant checks, verify UTF-8, confirm only intended files changed
- [ ] Wait for review — do not commit unless explicitly requested

---

## Hackathon Verification

- [ ] Is this a hackathon task?
- [ ] Is MVP sufficient?
- [ ] Can scope be reduced?
- [ ] Is architecture preserved?
- [ ] Is documentation still correct?
- [ ] Is validation completed?
